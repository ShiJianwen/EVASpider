我之前倒是做过一个开启的程序。思路是这样的：当切换上面的分类的时候在生成一个大类的cookie，同时触发ajax，将内容返回到下面的div里面。
而分页是公共的，每次点击换页的时候，首先获取cookie，判断是哪个分类，然后再通过ajax进行传值。
具体代码没有什么复杂的地方。

请问你每页显示的数据是在 单击页码的时候, 发 ajax 请求拿的当前页码的数据,
还是在 tab 激活的时候就已经通过 ajax 拿到当前分类中的所有数据, 然后只是客户端分页?
以及你每个tab下展示的效果都是一样的? 只是数据不一样?

根据楼主在评论中的回复, 现更新如下:

楼主可以先参考一下这个 网站 中的这一个模块中的代码.

红色箭头指向的,你可以认为是你的 Tab, 然后右侧的 黑色箭头指的, 你可以认为是页码.

然后整个JS的主要代码在这个地方(估且认为它是一个类吧): http://webresource.c-ctrip.com/ResFltIntlOnline/R10/Booking/online/package/service/index/AjaxTabService.js

然后实例化这个类的代码为:

var isInit = false,
    goback = online.$c('service.index.AjaxTabService', {//往返特惠
        box: $('#gobak_airline'),
        item: 'p.airline_right a',
        content: 'div.goback_content',
        style: 'current',
        MaxTryTimes: 1,//最大重试次数1次
        IntervalTime: 3000,//每隔3秒重试一次
        getName:function(item){
            return [this.box.find('span.roundtrip_depart').attr('name'), item.attr('name')].join('__');
        },
        getUrl:function(item){
            if(isInit){//后端要求第一次不要传DCity,以便识别ip,故做此处理 =,=
                return '/international/tools/HomeHandler.ashx?type=RT&DCity='+this.box.find('span.roundtrip_depart').attr('name')+'&Region='+item.attr('name');
            }
            return '/international/tools/HomeHandler.ashx?type=RT&Region=' + item.attr('name');
        },
        ajaxCallBack:function(target, ret){
            ret = $.parseJSON(ret);

            if(ret === null){//返回的内容不正确时
                if(this.checkTryLimit(target)){//检测是否可以允许重试
                    return;//因为新建出来的,本身显示的状态就是Loading,所以不需要再做其他的事情了
                }
                target.remove();//移除新建出来的节点
                this.box.find('.airline_busy').css('display', '');
                //this.tmpl.css('display', '');//将Loading重新显示出来,以便下次单击时重新去读取
                return;
            }

            if(isInit === false){
                isInit = true;
                var a = $('ul.airline_list a');
                for(var i=a.length; i--;){
                    if(a[i].name === ret.DepartCity){
                        var t = $('div.airline_box:first span.roundtrip_depart');
                        t.html(a[i].innerHTML + (online.charset === 'gb2312' ? '出发' : '出發'));
                        t.attr('name', a[i].name);
                        t.attr('rname', a[i].innerHTML);
                        break;
                    }
                }
                target[0].className = target[0].className.replace(/(...)__(\d)/, ret.DepartCity + '__$2');
            };

            target.html($.tmpl.render('<ul class="goback_list clearfix">\
                {{each(k,v) FlightList}}\
                <li data-ubt="Dupli_WhereGo_Detail${k+1}">\
                    <a href="${SearchURL}" target="_blank" style="z-index:7;">\
                    <span class="arrow"></span>\
                    <h4 class="takeoff"><span class="depart">${DCityName}</span><b class="ico_flight"></b>{{if IsTransfer}}<span class="transit">中转</span>{{/if}}</h4>\
                    <h4 class="arrive">${ACityName}</h4>\
                    <p>${online.toShortDate(GoDate)}<b class="ico_goback"></b>${online.toShortDate(BackDate)}</p>\
                    <p>往返：<strong class="base_price"><dfn>&yen;</dfn>${online.toMoney(Price)}</strong> 起</p>\
                    <p class="flight_name">\
                    <span class="flight_logo pubFlights_${AirlineCode}">${AirlineName}</span>\
                    </p>{{if ClassTypeName != ""}}<span class="tag">${ClassTypeName}</span>{{/if}}\
                    </a>\
                </li>\
                {{/each}}\
                </ul>', ret));

            target.find('ul.goback_list li:gt('+(ret.FlightList.length > 8 ? 8 : 5)+')').addClass('hide_on_small');

            target.find('li').bind('mousedown', function(e){
                $(e.target).attr('data-ubt', $(this).attr('data-ubt'));
            });
        }
    });

不过上面的你能看明白的话,我就不用整DEMO了.

那个类, 你不用去看那个 create 是怎么实现的, 只用记下 它的第一个参数是给这个类起个名字,
第二个参数是这个 对象原型上的方法, 然后在 online.$c 这个方法 调用这个类的时候, 会自动调用 initialize 里的代码就可以了.

所以 直接从 initialize 中看起就可以了.

与你的需求不同之处是, 这里做为页码的部分是 固定的, 而你的是 根据 返回的内容决定的, 但这个基本上不是问题, 只需要在 第一次获取到数据之后, 生成一下页码 更新上去, 再绑定事件(或者上层用委托的话,就不用再绑定事件了).
